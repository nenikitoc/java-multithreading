### Создание Thread и Runnable
__Thread__ — класс, представляющий поток исполнения. Создается путем наследования.  
__Runnable__ — интерфейс, описывающий задачу (логику). Реализация Runnable предпочтительнее: она гибче, не занимает единственное возможное наследование в Java и разделяет логику задачи от управления потоком.  
__Thread.start()__ - метод, который запрашивает у JVM создание нового системного потока и запускает в нем метод run().  

### Join и isAlive
__Thread.join()__ - приостанавливает выполнение текущего потока до тех пор, пока поток, у которого вызван `join()`, не завершит свою работу. Позволяет упорядочить выполнение задач.  
__Thread.isAlive()__ - возвращает true, если поток уже был запущен (`start()`) и еще не завершил работу.  

### Synchronized
__Synchronized__ - ключевое слово для создания критических секций. Гарантирует, что только один поток может выполнять защищенный код в конкретный момент времени.  
__Блок__ - управляет (блокирует) определенным блоком кода в методе.

```Java
private final Object lock = new Object();

public void method(){
    // some non-lock logic
    synchronized(lock){
        // lock logic
    }
}
```
__Синхранизированный метод__ - управляет (блокирует) всем методом
```Java
public synchronized void method(){
    // lock logic
}
```
__Static метод__ - управляется (блокируеться) весь класс
```Java
public class User{
    static synchronized void staticMethod(){
        // lock logic
    }
}
```
Проблема: Чрезмерное использование `synchronized` превращает многопоточную программу в последовательную (потоки выстраиваются в очередь). Рекомендуется блокировать как можно меньше кода.  

### Wait и Notify
Методы класса `Object`, используемые для координации потоков внутри `synchronized` секций. Позволяют одному потоку ждать сигнала от другого.  
* __wait()__ — поток «отпускает» монитор (ключ) и засыпает. Это позволяет другим потокам войти в синхронизированный блок.
* __notify()__ — пробуждает один случайный поток, который ранее вызвал `wait()` на этом же объекте.
* __notifyAll()__ — пробуждает все ожидающие потоки.

```java
synchronized (lock) {
    while (!ready) { 
        lock.wait(); // waiting signal
    }
    // some code where (ready == true) and ().notify())
}
```

### Volatile
__Volatile__ - модификатор поля, гарантирующий видимость изменений. Переменная читается и записывается напрямую в основную память (RAM), минуя кэши ядер процессора. Также запрещает переупорядочивание (reordering) инструкций.  
Важно: Не обеспечивает атомарность сложных операций (например, i++).  

```Java
private volatile int x;
private volatile boolean flag;
private volatile Object object;
``` 

### Atomic
__Atomic__ (пакет java.util.concurrent.atomic) — классы для безопасных операций над переменными без использования тяжелых блокировок (synchronized).  
Основаны на механизме CAS (Compare-And-Swap): процессор обновляет значение только в том случае, если оно не изменилось с момента чтения. Если изменилось — операция повторяется в цикле.  
* AtomicInteger, AtomicLong, AtomicBoolean - для примитивов.
* AtomicReference - безопасная ссылка на объекты.
